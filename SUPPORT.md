# Support

## Usage questions

Open a GitHub issue with:

- command(s) you ran
- expected behavior
- actual behavior/output
- environment (`dev` or `prod`)

## Bug reports

Open a GitHub issue and include:

- minimal reproduction steps
- exact CLI version (`skillmd --version`)
- Node version (`node --version`)
- relevant logs/output (redact tokens and secrets)

## Security issues

For vulnerabilities, use private reporting only:

- see [SECURITY.md](SECURITY.md)

## Scope

Support is best-effort for released CLI behavior and documented strict-v1 contracts.
